# HW1: Lead Capture to CRM Pipeline
## Overview
This report documents the implementation of the Lead Capture pipeline utilizing the Antigravity workflow system. The resulting solution maps an incoming webhook, transfers the lead data safely via service-account to a real Google Sheet, and dynamically generates a personalized AI summary.

## 1. How the Solution Meets the Requirements
Our strategy ensures no hardcoded keys sit in the codebase and real production pathways are utilized instead of the initial Mock CRM placeholders:

- **Trigger (`WebhookTrigger`)**: The `index.js` orchestrator hosts an Express endpoint listening at `POST /api/webhook/lead-capture`, awaiting `{name, email, message}` correctly formatted payloads from the system trigger node.
- **Processing Function (`DataMapper`)**: Through template string interpolation (`{{ incoming.body.name }}`), the script parses the payload and applies the custom server `$timestamp` properly mapping data to the `node-2-processing` states.
- **Google Sheets Integration (`GoogleSheetsConnector`)**: We eliminated the mock RestApiConnector and configured a `GoogleSheetsConnector`. The `googleapis` Node module securely uses the service account JSON (`GOOGLE_SERVICE_ACCOUNT_PATH`) and Sheet ID (`GOOGLE_SHEETS_ID`) retrieved via `$env` paths, pushing the mapped variables directly to `Sheet1`.
- **AI Completion (`LLMCompletion`)**: Implementing the `@google/generative-ai` Gemini SDK, the workflow uses `GEMINI_API_KEY` to send a prompt referencing `leadName` and `leadMessage`. The API contextually outputs a reliable 1-2 sentence lead summary into the final process block.

## 2. Workflow Structure
Structured internally as a directed JSON graph, the executor traverses `workflow.json` mapping dependencies sequentially. Definitions are populated dynamically using configurations written in `logic.json`.

### Sequence and Execution Order
`node-1-trigger` → `node-2-processing` → `node-3-google-sheets` → `node-4-ai` → `node-5-end`

### Inputs / Outputs Per Node
**1. WebhookTrigger (`node-1-trigger`)**
- **Input**: External HTTP POST Request containing the user information.
- **Output**: JSON Body bound to workflow state tree under `incoming.body`.

**2. DataMapper (`node-2-processing`)**
- **Input**: The unformatted HTTP payload block alongside dynamic time inputs.
- **Output**: `leadName`, `leadEmail`, `leadMessage`, `receivedAt` populated in scope.

**3. GoogleSheetsConnector (`node-3-google-sheets`)**
- **Input**: Google authentication properties referenced via environment strings plus output mapped items.
- **Output**: Physical validation of data inserted into `Sheet1` externally. Continues tree on success.

**4. LLMCompletion (`node-4-ai`)**
- **Input**: Synthesized textual prompt "You are a sales assistant..." replacing mapped attributes.
- **Output**: The summarized text generated by Gemini-1.5-pro, logged directly inside orchestrator state.

**5. SuccessTerminator (`node-5-end`)**
- **Input**: Signals completed processes.
- **Output**: Sends an HTTP `200 Success` output to the caller, containing the final AI-generated summary object.
